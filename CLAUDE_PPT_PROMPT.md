# Prompt for Claude: Create Outstanding Attractive PowerPoint

```
You are an expert presentation designer specializing in accessibility-first design. 
Create an outstanding, visually attractive PowerPoint presentation for "NeuroVidya - 
A Dyslexia-First Adaptive Learning Platform."

PROJECT OVERVIEW:
NeuroVidya is a comprehensive web-based learning platform designed specifically for 
individuals with dyslexia. It features a dyslexia-first design methodology, AI-powered 
personalization, and WCAG 2.1 AA compliance.

PRESENTATION REQUIREMENTS:

1. TOTAL SLIDES: 12
   - Slide 1: Team Name Page (Title)
   - Slide 2: Introduction
   - Slide 3: Literature Review
   - Slide 4: Problem Statement
   - Slide 5: Objectives
   - Slide 6: Software Requirements
   - Slide 7: Methodology
   - Slide 8: Implementation
   - Slide 9: References
   - Slide 10: Application
   - Slide 11: Conclusion
   - Slide 12: Thank You / Q&A

2. DYSLEXIA-FRIENDLY DESIGN PRINCIPLES:

   FONTS:
   - Primary font: Lexend (or OpenDyslexic, Arial)
   - Header size: 44-54pt
   - Body text size: 24-32pt (minimum 24pt)
   - Letter spacing: 0.05em
   - Word spacing: 0.1em
   - Line height: 1.5-1.8
   - NEVER use italics or underlining
   - Use bold sparingly for emphasis only
   - Left-align all text (no center alignment for body)

   COLORS (Research-Based Palette):
   - Background: #FAF6F0 (cream - reduces visual stress)
   - Primary text: #2D3748 (dark gray - high contrast)
   - Accent Blue: #3B82F6 (calm blue for headers)
   - Accent Purple: #8B5CF6 (for variety)
   - Highlight Yellow: #FBBF24 (for emphasis)
   - Success Green: #10B981 (for completed items)
   - Red: #EF4444 (for problems/challenges - use sparingly)
   
   CONTRAST REQUIREMENTS:
   - All text must have minimum 4.5:1 contrast ratio
   - Headers should have 7:1+ contrast
   - Use color PLUS text/icons for information (never color alone)

   LAYOUT PRINCIPLES:
   - Maximum ONE main idea per slide
   - Use 50% white space (avoid clutter)
   - Left-align everything
   - No columns when possible
   - Wide margins on all sides
   - Consistent header/footer on all slides
   - Page numbers on all slides except title

3. VISUAL ELEMENTS TO INCLUDE:

   SLIDE 1 (Title):
   - Large, bold title "NEUROVIDYA"
   - Subtitle with icon
   - Team names with placeholder photos
   - Institution logo placeholder
   - Date/academic year

   SLIDE 2 (Introduction):
   - Icon for each key point (bulb, users, chart, heart)
   - Simple 4-quadrant layout or vertical flow
   - Maximum 4 key points

   SLIDE 3 (Literature Review):
   - Comparison table visual (existing solutions vs NeuroVidya)
   - Color-coded: Green for strengths, Red for limitations
   - Simple icons for each theoretical framework

   SLIDE 4 (Problem Statement):
   - Before/After comparison visual
   - Problem icons in red, solution in green
   - Large central question with emphasis
   - Simple bullet points with checkmarks/X marks

   SLIDE 5 (Objectives):
   - Checkmark icons (✅) for completed objectives
   - Progress bars for each section
   - Targets vs Current comparison table
   - Icons for each objective type (🎯 primary, 📊 secondary)

   SLIDE 6 (Software Requirements):
   - Icon grid for different requirement categories
   - Platform compatibility table with checkmarks
   - System requirements in visual boxes
   - Browser logos/representations

   SLIDE 7 (Methodology):
   - Timeline visualization (horizontal or vertical)
   - Phase boxes with ✅ for completed, 🔄 for in progress
   - Week indicators for each phase
   - Simple flow arrows between phases

   SLIDE 8 (Implementation):
   - System architecture diagram (box diagram with layers)
   - Component count in visual circles
   - Performance metrics as progress bars or gauges
   - Code snippet placeholder (small, styled)

   SLIDE 9 (References):
   - Categorized lists with icons
   - Book icon for books, 📄 for papers, 🔗 for online
   - Numbered lists, organized by category
   - Keep to maximum 8-10 key references

   SLIDE 10 (Application):
   - Use case illustrations with simple icons
   - For whom sections (students, educators, admin)
   - Timeline for future enhancements (short-term, long-term)
   - Deployment platform logos/boxes

   SLIDE 11 (Conclusion):
   - Summary infographic (key stats in circles)
   - Three-section layout: Achievements, Impact, Vision
   - Call to action in highlighted box
   - Contact information with icons

   SLIDE 12 (Thank You):
   - Large centered "Thank You"
   - Contact information with icons
   - QR code placeholder for demo link
   - Social media icons
   - Acknowledgments section

4. DESIGN CONSISTENCY:

   On EVERY slide (except title):
   - Top: Slide number in corner (right-aligned)
   - Bottom: Progress indicator (slide X of 12)
   - Consistent header font and size
   - Consistent bullet point style (custom icon bullets)
   - Footer with project name (small)

5. TRANSITIONS AND ANIMATIONS:
   - Use ONLY simple fade transition between slides
   - Subtle entrance animation for bullet points (fade in, 0.3s each)
   - NO distracting animations, spins, or bounces
   - Consistent timing across all slides
   - All content visible within 2 seconds of slide appearing

6. SPECIAL ACCESSIBILITY FEATURES:
   - All images have descriptive alt text
   - Charts use patterns + colors (not just colors)
   - Tables have clear headers
   - Lists use custom icon bullets, not just dots
   - Simple, predictable navigation
   - High contrast mode compatible

7. OUTPUT REQUIREMENTS:
   - Create as PowerPoint .pptx file
   - All fonts embedded
   - All design elements applied
   - Ready to present immediately
   - Include speaker notes for each slide
   - Export as PDF option available

8. PRESENTATION FLOW:
   - Each slide should take 2-3 minutes to present
   - Total presentation time: 25-30 minutes
   - Include pause points in speaker notes
   - Suggest questions to ask audience at specific slides

9. BRANDING ELEMENTS:
   - If available, use NeuroVidya logo
   - If not available, create text-based logo with icon
   - Consistent color scheme throughout
   - Professional, modern appearance
   - Academic yet approachable tone

10. QUALITY CHECKLIST:
   ✅ All text readable at 100% zoom
   ✅ High contrast maintained throughout
   ✅ No color-only information conveyance
   ✅ Maximum 6 bullet points per slide
   ✅ One main idea per slide
   ✅ Icons/images used for visual support
   ✅ Consistent design language
   ✅ Professional appearance
   ✅ Dyslexia-friendly principles applied
   ✅ Ready to present without modifications

CONTENT TO USE:
Use the detailed content from "NeuroVidya_PPT_Content.md" for all slide content.
Extract key points and present them visually - don't overcrowd slides.

Remember: This presentation is ABOUT dyslexia accessibility, so it must MODEL 
dyslexia accessibility in its design. The presentation itself should be an 
example of dyslexia-friendly design principles.

Create the PowerPoint now with all specifications applied.
```

## Additional Instructions for Best Results:

**If using PowerPoint Designer:**
1. After creating slides, use Design Ideas for layout suggestions
2. Choose layouts that follow dyslexia-friendly principles
3. Avoid Designer suggestions that violate accessibility rules

**If using Canva:**
1. Use "Education" or "Academic" templates
2. Customize colors to match the specified palette
3. Replace fonts with Lexend or OpenDyslexic
4. Export as PowerPoint .pptx format

**If using Google Slides:**
1. Download as PowerPoint .pptx
2. Check all formatting transfers correctly
3. Verify embedded fonts work
4. Test on presentation computer

**Final Testing Checklist:**
- [ ] Test on presentation computer beforehand
- [ ] Verify all fonts load correctly
- [ ] Check colors on projector/screen
- [ ] Test presenter notes visibility
- [ ] Practice with slide advances
- [ ] Have backup PDF version
- [ ] Print handouts in dyslexia-friendly format

---

it's presenting about - dyslexia-friendly, accessible, and inclusive design.**
